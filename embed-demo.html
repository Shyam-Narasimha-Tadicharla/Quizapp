Data-attribute embed

Simplest method — no JavaScript required

Add data-quiz-engine to any <div> and paste your questions into data-questions. The script auto-discovers it on page load.

A) First option
B) Second option
C) Third option

JavaScript init

For developers who build questions dynamically

Use QuizEngine.init() when you're fetching questions from an API, a database, or building them at runtime. Pass a pre-parsed array or raw text — either works.

// Option A — pass raw text (engine parses it for you) QuizEngine.init({ target: "#my-quiz", title: "Onboarding Check", text: `Q: What is the capital of France? A) Berlin B) Madrid C) Paris D) Rome Correct: C` }); // Option B — pass a pre-parsed array (e.g. from your own API) QuizEngine.init({ target: "#my-quiz", title: "Onboarding Check", questions: [ { q: "What is the capital of France?", opts: ["Berlin", "Madrid", "Paris", "Rome"], correct: 2 // zero-indexed } ] });

Listening to results

Send scores to your backend when a user submits

When a quiz is submitted, the host element fires a quizComplete custom event. Listen to it and do whatever you need — POST to your API, update a leaderboard, show a certificate.

const quizEl = document.querySelector("#my-quiz"); quizEl.addEventListener("quizComplete", (e) => { const { score, total, percent, answers } = e.detail; // Example: send to your own API fetch("/api/results", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ score, total, percent, answers }) }); });
// Payload shape (e.detail) { score: 4, // number of correct answers total: 5, // total questions percent: 80, // percentage correct answers: [ { question: "What does CSS stand for?", chosen: "Cascading Style Sheets", correct: "Cascading Style Sheets", isCorrect: true }, // … one entry per question ] }

Complete answer breakdown — you have everything you need to store per-user results, flag weak areas, or trigger follow-up content.
